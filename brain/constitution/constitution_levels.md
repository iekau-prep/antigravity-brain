Purpose

本書は、

家買う予備校 Constitution Library 全体の
変更レベルを定義する。

本書は思想を定義しない。

思想をどのレベルで管理するかのみ定義する。

⸻

Change Levels

Level A｜憲法

家買う予備校そのものを規定する思想。

変更には、

* 思想監査室レビュー
* Product Owner承認

を必要とする。

対象例

・家買う予備校の目的
・decision定義
・STATE定義
・判断形成思想
・continuity思想
・loop思想
・fixed_core思想
・現実との接触思想
・受渡思想
・チャネル思想
・recommendation境界

⸻

Level B｜準憲法

思想を維持したまま、

構造改善可能なもの。

Builder提案可。

思想監査室確認。

Product Owner採用判断。

対象例

* module責務
* module受渡
* CTA責務
* Builder Flow
* Reading Guide

⸻

Level C｜設計

思想へ影響しない実現方法。

対象例

* API
* Schema
* URL
* UI
* Database
* 実装方式

⸻

Level D｜運用

完成後も継続改善される運用。

対象例

* KPI
* 観察
* Phase
* 公開
* 通知タイミング
* 運用フロー

⸻

AI Governance

Builder

* Level Aを変更しない
* Level Bを提案する
* Level Cを設計する
* Level Dは番人と連携する

思想監査室

* Level Aを監査する
* Level Bを確認する

技術監査

* Level Cのみ担当する

番人

* Level Dのみ担当する

Product Owner

* 全Levelの最終採用判断を行う。

⸻

Basic Principle

重要。

変更禁止ではない。

変更ハードルが異なる。

Levelが高いほど、

レビューと承認が必要になる。
